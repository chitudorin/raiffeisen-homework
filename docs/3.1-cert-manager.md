# cert-manager

For extra style points (and because I already like this setup I might as well use it for my homelab) I'll also set up cert-manager with a wildcard certificate.

I added an A record in the form of `*.home.domain.tld` pointing to my home network, will use cert-manager to issue a wildcard certificate and (hopefully) set up ingress-nginx to use this certificate automatically so I don't have to write it manually (the less YAML I have to write, the less chance I might mess something up).

I'm using the official documentation for installing with Helm and I can pretty much just copy the ingress-nginx setup to make sure everything's fine.

Turns out there is a difference compared to nginx-ingress and that's CRDs. The documentation mentions `--set crds.enabled=true` flag, this translates to:

```
spec:
  chart:
    spec:
      values:
        crds:
          enabled: true
```
